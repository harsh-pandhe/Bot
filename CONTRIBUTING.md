# Contributing to MazeSolver Robot

Thank you for your interest in contributing! Here are some guidelines to help you get started.

## How to Contribute

### Reporting Issues
- Check existing issues before creating a new one
- Provide detailed descriptions of bugs or feature requests
- Include relevant hardware/software versions
- Add sample data or logs when applicable

### Submitting Changes

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Keep commits focused and well-documented
   - Test thoroughly before submitting
4. **Push to your fork**
5. **Submit a Pull Request**
   - Describe what changes you made and why
   - Reference any related issues

## Development Guidelines

### Code Style
- **Python**: Follow PEP 8
- **Arduino/C++**: Use consistent indentation (2 spaces)
- Add comments for complex logic
- Keep functions focused and modular

### Testing
- Test new features with real hardware when possible
- Validate neural network training with multiple datasets
- Document test procedures

### Documentation
- Update README if adding features
- Add docstrings to Python functions
- Comment firmware changes clearly
- Update CHANGELOG.md

## Areas for Contribution

- **Algorithms**: Improve maze-solving strategies
- **Hardware**: Support for additional sensors/platforms
- **ML**: Better training pipelines and preprocessing
- **Documentation**: Clearer guides and examples
- **Testing**: Add test cases and benchmarks

## Questions?

Open an issue with the `question` label or start a discussion.
